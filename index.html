38. Merge Set

Input: {1,2},{2,3},{3,4}{5,6,7},{7,8},{9},{9}
Output: {1,2,3,4},{5,6,7,8},{9}
能不能够维护一个hashmap<num, set_id> 比如1->{1,2}
同时维护一个map存set间关系
碰到{2, 3}时找2和3没有没在hashmap中，发现2在，
则将{2, 3}的father标记为{1,2}，。。。最后从每个点向父亲走并合并set内容
如果有N个set，平均长度M，建立set关系要NM，合并需要NM

37. Dominoes

骨牌是不是可以直接存2个int 组合起来的string..
找的时候..直接reverse下..看看那个string在不在map里.

每个牌上有两个数字，上下各一个，都在1-6之间，要找出有没有两个match的骨牌，
match的定义是两组数加起来都是6，比如一个骨牌是（1,5），
那么match的骨牌是（1,5）或者（5,1），总之相当弱智的一题，
用哈希表解决，存的时候存个pair，找的时候找两下就行了。

31. Deepest Common Ancestor

思路路：BFS找 deepest leaf node 的list。然后他们的lowest common ancestor
其实就是第一个leaf和
最后一个leaf的lowest common ancestor 然后按类似LC236那么写就可以了了。

edge case： 假如最深叶节点只有一个的话，你这个方法是不是还要特殊处理一下？
思路：我来说一个思路，先traverse一遍,
把每一个点的深度和父亲存在hash表里面，然后把最深得
那些点放在queue里面做个向父亲节点的bfs，
队列的size会不不断变小，当只有一个点的时候就是答案啦
